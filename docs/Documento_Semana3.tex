\section{Objetivo de Semana 3}

El objetivo de esta fase fue cerrar el ciclo experimental de modelado con una comparacion formal y trazable entre modelos de clasificacion binaria, usando el mismo protocolo para todos los candidatos. Esta semana se enfoco en cuatro frentes:

\begin{enumerate}[leftmargin=1.2cm]
\item Validacion experimental robusta y comparable.
\item Afinacion de hiperparametros para Regresion Logistica (LR), KNN y SVM-RBF.
\item Analisis de ajuste (sobreajuste/subajuste) y consistencia CV vs test.
\item Seleccion final del mejor modelo con justificacion tecnica explicita.
\end{enumerate}

La prioridad operativa del problema es clinica: minimizar falsos negativos de casos malignos. Por eso, durante el tuning la metrica objetivo principal fue el recall de la clase positiva (\texttt{Class = 4}).

\subsection{Escenarios de variables}

Se evaluaron dos escenarios para mantener comparacion formal:

\begin{itemize}[leftmargin=1.2cm]
\item Escenario A: todas las variables.
\item Escenario B: top 3 variables (\texttt{Bare Nuclei}, \texttt{Uniformity of Cell Size}, \texttt{Uniformity of Cell Shape}).
\end{itemize}

\subsection{Esquema de validacion aplicado}

\begin{itemize}[leftmargin=1.2cm]
\item \texttt{StratifiedKFold(n\_splits=5, shuffle=True, random\_state=42)}
\item Mismas metricas para todos los modelos base: accuracy, precision ponderada, recall macro, F1 ponderado y AUC OVR ponderado.
\item Mismo pipeline de preprocesamiento y entrenamiento por modelo.
\end{itemize}

Este esquema asegura condiciones experimentales identicas para LR, KNN y SVM.

\subsection{Control de fuga de informacion}

Se aplicaron reglas anti-leakage en todo el flujo:

\begin{enumerate}[leftmargin=1.2cm]
\item Escalado dentro de \texttt{Pipeline}.
\item Cada fold ajusta transformaciones solo con datos del fold de entrenamiento.
\item El test set no participa en tuning ni seleccion de hiperparametros.
\item La evaluacion en test se usa solo como verificacion final de generalizacion.
\end{enumerate}

\subsection{Sobreajuste/subajuste y sesgo-varianza}

En los modelos optimizados, los gaps train-valid son pequenos y clasificados como estables:

\begin{itemize}[leftmargin=1.2cm]
\item SVM A optimizado: gap = 0.0000
\item SVM B optimizado: gap = 0.0026
\item LR y KNN optimizados: gaps bajos en general
\end{itemize}

No se observa sobreajuste clasico fuerte. El trade-off dominante en SVM A optimizado es diferente:

\begin{itemize}[leftmargin=1.2cm]
\item Mejora extrema en sensibilidad (recall 1.0000, FN = 0)
\item Reduccion de precision y AUC frente a LR
\end{itemize}

Interpretacion tecnica:

\begin{itemize}[leftmargin=1.2cm]
\item Se reduce sesgo hacia la clase positiva (menos errores de omision de malignos).
\item Se acepta mayor costo de falsos positivos relativos.
\end{itemize}

Para el contexto del problema, este trade-off es tecnicamente razonable y defendible.

\section{Modelo final seleccionado y justificacion}

\subsection{Seleccion final}

\textbf{Modelo final de Semana 3: SVM optimizado (kernel RBF) en Escenario A (todas las variables).}

\subsection{Justificacion tecnica}

\begin{enumerate}[leftmargin=1.2cm]
\item \texttt{Recall\_Test\_maligna = 1.0000} (maximo posible).
\item \texttt{FN\_Test = 0}, criterio clinico dominante.
\item La mejora en validacion se sostiene en test (\texttt{Sostiene\_en\_test = si}).
\item Sin evidencia de sobreajuste fuerte en el analisis train-valid.
\end{enumerate}

\subsection{Trade-offs explicitos}

\begin{itemize}[leftmargin=1.2cm]
\item Frente a LR optimizada, SVM A sacrifica precision y AUC.
\item Se prioriza seguridad diagnostica (menos omisiones de malignos) sobre pureza probabilistica.
\item Esta decision es consistente con el objetivo del proyecto.
\end{itemize}
