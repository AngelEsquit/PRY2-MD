\section{Objetivo de Semana 2}
% ─────────────────────────────────────────────

Con base en los hallazgos del EDA de Semana 1, esta fase tuvo tres objetivos concretos: (1) convertir el dataset crudo en un dataset de modelado limpio y trazable; (2) implementar dos escenarios de variables como respuesta directa al hallazgo de colinealidad; y (3) establecer una línea base de rendimiento con tres modelos sin optimizar, usando las métricas prioritarias definidas en el EDA.

% ─────────────────────────────────────────────
\section{Preprocesamiento de Datos}
% ─────────────────────────────────────────────

\subsection{Diagnóstico del dataset original}

Se partió de \texttt{Datos.csv} con 683 registros y 11 columnas. Antes de entrenar cualquier modelo se realizó un diagnóstico sistemático del estado del dataset:

\midrule

\noindent\textbf{Nota sobre \texttt{Bare Nuclei}:} en la versión estándar del Wisconsin Breast Cancer Dataset publicada en UCI, esta variable contiene 16 valores codificados como \texttt{``?''}. En el presente dataset (\texttt{Datos.csv}), la columna ya viene limpia con valores numéricos válidos en el rango 1--10. Sin embargo, el pipeline de limpieza incluyó la detección y tratamiento de esos marcadores por robustez, por si aparecieran en re-ejecuciones con datos externos.

\subsection{Pasos de limpieza aplicados}

\begin{enumerate}[leftmargin=1.4cm]
\item \textbf{Eliminación de duplicados:} se identificaron y removieron 8 filas duplicadas exactas.
\item \textbf{Detección de marcadores inválidos:} búsqueda de valores \texttt{``?''} en todas las columnas; ninguno encontrado en este dataset.
\item \textbf{Verificación de rangos:} todas las variables predictoras confirmadas en el rango 1--10.
\item \textbf{Exclusión de identificador:} \texttt{Sample code number} excluido por no tener valor predictivo.
\item \textbf{Exportación:} se generó \texttt{Datos\_modelado.csv} como dataset oficial para modelado.
\end{enumerate}

\textbf{Sobre PCA:} se evaluó como alternativa para manejar la colinealidad, pero se descartó en esta etapa por tres razones: (i) solo hay 9 variables originales, lo que limita la compresión significativa; (ii) se priorizó interpretabilidad clínica de variables observables; (iii) la estrategia L2 + comparación A/B permite controlar la colinealidad sin perder trazabilidad de cada rasgo citológico. Si el rendimiento en Semana 3 no fuera satisfactorio, PCA quedaría como alternativa para Semana 4.

% ─────────────────────────────────────────────
\section{Modelos Base}
% ─────────────────────────────────────────────

Se entrenaron tres modelos sin optimización de hiperparámetros para establecer una línea de referencia. Todos usan \texttt{StandardScaler} dentro de un \texttt{Pipeline} para evitar fuga de información.

\subsection{Regresión Logística (LR)}

Modelo lineal de referencia para clasificación binaria. La regularización L2 por defecto (\texttt{C=1.0}) se usó \textit{deliberadamente} como respuesta técnica al riesgo de colinealidad identificado en el EDA: el término de penalización estabiliza los coeficientes cuando las variables son altamente correlacionadas, evitando que pequeñas perturbaciones en los datos generen cambios grandes en los pesos del modelo.

\textbf{Parámetros base:} \texttt{solver=`lbfgs'}, \texttt{penalty=`l2'}, \texttt{C=1.0}, \texttt{class\_weight=`balanced'}.

\subsection{K-Nearest Neighbors (KNN, k=5)}

Clasificador no paramétrico basado en distancia. No asume ninguna forma funcional para la frontera de decisión, lo que lo hace útil como punto de comparación cuando la distribución real de los datos es incierta. Requiere escalado de variables (realizado en el pipeline). Se usó $k=5$ como baseline estándar.

\textbf{Parámetros base:} \texttt{n\_neighbors=5}, \texttt{weights=`uniform'}, \texttt{metric=`minkowski'}.

\subsection{SVM con kernel RBF}

Modelo no lineal basado en maximización de margen. Se eligió por dos razones derivadas del EDA: (1) la alta separabilidad visual entre clases sugería que una frontera de margen máximo sería eficiente; (2) el kernel RBF permite capturar relaciones no lineales en el espacio de variables sin imponer forma funcional explícita. Adicionalmente, SVM es intrínsecamente robusto frente a outliers por construcción (solo los vectores de soporte influyen en la frontera).

\textbf{Parámetros base:} \texttt{C=1.0}, \texttt{gamma=`scale'}, \texttt{kernel=`rbf'}, \texttt{class\_weight=`balanced'}.

El patrón es consistente: todos los modelos pierden recall al reducir variables, pero LR pierde 4 veces más que KNN. Esto tiene dos implicaciones para Semana 3:

\begin{enumerate}[leftmargin=1.4cm]
\item El \textbf{Escenario A} es el candidato principal para la optimización final, ya que sostiene mejor rendimiento en todos los modelos.
\item LR necesitará \textbf{tuning de regularización} para estabilizarse, especialmente si se expande el espacio de hiperparámetros más allá del valor por defecto ($C=1.0$).
\end{enumerate}

% ─────────────────────────────────────────────
\section{Conclusiones de Semana 2}
% ─────────────────────────────────────────────

\begin{enumerate}[leftmargin=1.4cm]
\item El dataset fue limpiado de forma trazable: 8 duplicados eliminados, identificador excluido, 675 registros finales.
\item \textbf{SVM con kernel RBF} es el mejor modelo base en Escenario A, con el mayor Recall (0.9574) y el menor número de falsos negativos (2).
\item LR y KNN empatan en Escenario A, pero LR es más sensible a la reducción de variables, lo que confirma el riesgo de colinealidad identificado en Semana 1.
\item El Escenario A supera al B en todos los modelos. Escenario B no es un fracaso: evidencia que con solo 3 variables ya se obtiene 91--93\% de recall, lo que cuantifica el poder predictivo del subconjunto discriminativo.
\item La hipótesis H3 (Recall como métrica principal) queda reforzada: un modelo con accuracy de 95.6\% puede aún tener 3 falsos negativos clínicamente críticos.
\item La hipótesis H4 (impacto de colinealidad en LR) queda confirmada: LR es el modelo más afectado por la reducción de variables.
\end{enumerate}
